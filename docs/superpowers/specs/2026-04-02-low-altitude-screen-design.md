# 低空管理大屏设计说明

日期：2026-04-02

## 1. 页面定位

本页面定位为：

- 低空管理大屏
- 领导汇报优先
- 以飞行任务态势为主线
- 用全局总览方式呈现任务、无人机、航线、告警、飞手五类信息

这不是一个重交互的调度工作台，也不是普通后台首页，而是一块以全局态势展示为主的汇报屏。

页面核心目标只有三点：

- 让观看者一眼看清当前有哪些飞行任务正在运行
- 让观看者快速判断无人机在线与保障情况
- 让观看者同步掌握当前全局重点告警

## 2. 整体布局

页面采用 `全屏 3DGS 场景 + 左右悬浮信息层` 的结构。

### 2.0 路由与目录约束

本页面不再放在 `dashboard` 目录之下，而是作为独立页面建设。

- 页面路由使用 `/low-altitude-screen`
- 页面目录使用 `src/views/low-altitude-screen/`
- 该目录与 `src/views/dashboard/` 平行
- 不再把本页面作为 `dashboard` 的子实现或壳层挂载

### 2.1 中间主视觉

- 中间区域由 3DGS 场景或卫星/地形底图覆盖整个屏幕
- 地图/场景是唯一主背景，不再用顶部横条或标题卡片切割画面
- 左右信息面板悬浮在场景之上，而不是把地图切成中间一块卡片

### 2.2 左侧悬浮面板

左侧只保留一个主模块：

- `任务态势`

原先分开的 `今日飞行任务数` 和 `执行中任务` 已合并进这个模块头部，避免信息重复。

### 2.3 右侧悬浮面板

右侧保留两个模块：

- `无人机在线态势`
- `重点告警`

原先分开的 `重点告警数` 已合并进 `重点告警` 模块头部，避免信息重复。

## 3. 模块功能清单

### 3.1 中间 3DGS 场景

中间场景承担全局态势展示，不承担复杂联动逻辑。

### 展示内容

- 飞行任务对应的主航线
- 无人机点位
- 飞手点位
- 告警点位
- 必要的区域态势标识

### 展示规则

- 默认只显示点位或图标
- 不默认铺满文字标签
- 选中某个点位后，再展开局部文字标签
- 主图以全局总览为主，不做频繁聚焦切换

### 已明确的交互边界

- 左侧任务点击后，`不驱动` 中间场景自动聚焦
- 右侧告警模块始终显示 `全局告警`，不随左侧任务切换
- 中间场景保持全局态势，不做工作台式联动

### 当前未定项

- 某个任务被选中后，是否显示任务覆盖区域
- 当前阶段先不强行定义，后续实现时按“简洁优先”处理

### 3.2 左侧模块：任务态势

这是左侧唯一主模块，也是页面的主叙事入口。

### 模块目标

- 先给出任务层面的全局摘要
- 再给出当前执行中的重点任务列表
- 让观看者知道“今天有多少任务”“现在在做什么”

### 模块头部摘要

只保留两个数字：

- `今日任务`
- `执行中`

不再单独拆出独立卡片。

### 模块主体

主体为 `执行中任务列表`，支持自动滚动。

### 每条任务卡片字段

每条任务卡片只保留四个字段：

- `任务名称`
- `执行状态`
- `当前航线`
- `开始时间`

### 任务列表展示建议

- 默认展示当前可视范围内的若干条任务卡片
- 超出部分自动滚动
- 任务名称为主标题
- 执行状态用状态标签突出
- 当前航线与开始时间放在次级信息层

### 3.3 右上模块：无人机在线态势

这是资源态势模块，不做成厚重的完整资源列表。

### 模块目标

- 告诉观看者当前无人机整体在线情况
- 补充少量重点在线无人机，增强信息价值
- 不把右上做成另一个后台业务列表

### 模块头部摘要

建议保留三项摘要：

- `在线`
- `可调度`
- `待命`

### 模块主体

主体保留 `3 条无人机列表`

### 每条无人机列表信息

建议每条包含：

- 无人机编号或名称
- 当前任务或当前状态
- 一个简洁状态标签

这部分的重点是“看态势”，不是“查完整资源台账”。

### 3.4 右下模块：重点告警

这是右侧主风险模块，用于持续播报全局重点异常。

### 模块目标

- 先说明当前全局告警压力
- 再播报 3 条最值得关注的重点告警
- 让右侧保持“全局风险播报栏”的角色

### 模块头部摘要

建议合并显示：

- `总数`
- `高等级`
- `跟踪中`

### 模块主体

主体固定展示 `3 条简明告警`

### 每条告警卡片字段

每条告警卡片只保留三个字段：

- `告警名称`
- `告警等级`
- `发生时间`

不再塞入关联任务、区域、处置状态等冗余文字，避免右侧过重。

## 4. 已明确的视觉规则

当前脑暴已经收敛出的视觉规则如下：

- 不要顶部信息展示条
- 不要独立标题卡片
- 左右信息层悬浮在全屏场景之上
- 中间场景必须覆盖整屏
- 底图方向使用3DGS 模型场景
- 页面整体以“干净、完整、可远看”为优先

## 5. 实施边界

为了后续实现不跑偏，以下内容暂时不纳入第一版实现重点：

- 顶部时间、天气等辅助信息
- 左右边栏与场景的复杂联动
- 左侧任务点击后驱动场景镜头跳转
- 右侧告警跟随左侧任务切换
- 大面积任务覆盖区的默认显示

第一版更重要的是先把以下骨架做稳：

- 全屏场景
- 左侧任务态势
- 右上无人机在线态势
- 右下重点告警
- 场景中的基础航线与点位表达
